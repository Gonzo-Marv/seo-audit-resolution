E.01

3 pages have duplicate meta descriptions

Error ✅ Resolved
About this issue. Semrush flags pages that share the same meta description. Duplicate descriptions make it harder for Google to differentiate pages in SERPs and often result in Google choosing its own snippet instead of yours.

Gave each of the three flagged static pages a unique, page-specific meta description. Unique descriptions are the baseline expectation for crawlable pages — they improve click-through rate and prevent Google from substituting its own snippet. Verified clear in the April 23 audit.

E.02

1 internal link is broken

Error ✅ Resolved
About this issue. An internal link points to a URL that returns a 4xx error. Broken internal links waste crawl budget, hurt user trust, and are a direct signal of site neglect.

Identified the broken link in a California retirement blog post and removed/replaced it. Same post contained the malformed href below, so both were addressed in the same pass.

E.03

1 link couldn't be crawled (incorrect URL format)

Error ✅ Resolved
About this issue. A link's href value was not a valid URL — crawlers can't resolve it and it produces a 404 for anyone who clicks.

Fixed a malformed <a> tag in the same California retirement blog post, where the href value was a sentence fragment rather than a URL. This was producing a second 404 on the same page and confusing crawlers that were trying to resolve it as a relative path.

W.01

1 page doesn't have a meta description

Warning ✅ Resolved
About this issue. A page has no meta description, so Google auto-generates one from on-page text.

Added a unique meta description to the /plan-and-price page.

W.02

48 pages don't have an H1 heading

Warning ✅ Resolved Caused a cascade
About this issue. H1 is the primary topical signal for a page. Missing H1s on 48 blog posts meant Google had to infer topic from the title tag and body copy alone.

Added an H1 at the Blog Template level, bound to each post's Name/Title CMS field. This template-level fix resolved all 48 flagged pages in a single change — every blog post now renders an H1 that matches its post title.

The fix shipped on April 21 and cleared this specific warning, but it surfaced a new one in the April 22 audit that's worth reading in tandem: because the title tag was also bound to the post name (without the brand suffix), H1 and title now rendered identical text on every post. See 49 pages have duplicate H1 and title tags for the full cascade narrative and how it was resolved.

Cascade

One template change resolved 48 missing-H1 warnings in a single action, but created 49 duplicate-H1/title warnings downstream. The resolution of the downstream issue (re-adding the brand suffix to the title tag) is documented in E.04 / W.06 equivalents — the trade-off for the fix was accepting longer title tags on some posts.

W.03

21 pages have too much text within the title tags

Warning ⚖️ Accepted
About this issue. Semrush flags title tags over 70 characters. Google truncates titles at roughly that length in SERPs but still considers the full text for ranking.

This issue had an unusual resolution path. The initial fix on April 21 stripped the " | Move Concierge" brand suffix from the Blog Template's SEO title binding, which brought 18 of the 28 flagged titles under the 70-character threshold. However, this fix had an unintended side effect: because the H1 was bound to the post name and the title tag was also bound to the post name (now without the suffix), all blog posts began rendering identical H1 and title text — which Semrush then flagged as 49 pages have duplicate H1 and title tags in the April 22 audit.

With Guy's guidance we judged that the duplicate H1/title flag was a higher-priority SEO concern than the title-length warning, and that the quickest route to resolution was to re-add the " | Move Concierge" suffix to the title tag. This textually differentiates the title from the H1 (matching Google's "same topic, different text" guidance) while accepting that some blog post titles will now exceed 70 characters. Google does not materially penalize over-length titles — it truncates them in SERPs but still considers the full text for ranking.

Decision

Better to have clearly distinct title/H1 pairs than perfectly-length-optimized duplicate ones. Taken deliberately: ranking guidance from Google points to topical alignment with lexical difference, which is exactly what the suffix restores. The truncation is a display concession, not a ranking concession.

W.04

10 pages have more than one H1 tag

Warning ✅ Resolved
About this issue. Best practice (and Google's recommendation) is one H1 per page. Multiple H1s dilute the topical signal.

Reduced to a single H1 per affected page on Home, /partners, the Services Template, MC Academy, and Privacy Policy. Other headings were demoted to H2. Matches W3C and Google's recommendation of one H1 per page.

W.05

Sitemap.xml not indicated in robots.txt

Warning ✅ Resolved
About this issue. robots.txt should contain a Sitemap: directive so crawlers can discover the sitemap without guessing its location.

This item was initially classified as blocked — we assumed editing robots.txt via Webflow required an Enterprise plan. That assumption turned out to be wrong, and the flag cleared on the April 23 audit through two related actions:

  1. Enabling Webflow's auto-generated sitemap. Marvin toggled on the auto-generate sitemap setting in Site Settings. When sitemap generation is on, Webflow automatically includes a Sitemap: directive in its auto-generated robots.txt — no Enterprise access required. This is useful learning: the "BLOCKED" assumption was wrong, and similar "needs Enterprise" claims should be verified before being treated as blockers.
  2. Submitting the sitemap to Google Search Console for the first time. No prior SEO engagement had ever submitted this site's sitemap to GSC. Done as part of this cycle, which also contributed to clearing the flag and — more importantly — unlocks the Coverage and Performance reports for ongoing measurement.
Takeaway

Webflow auto-manages robots.txt sitemap directives on non-Enterprise plans when sitemap generation is on. Future "needs Enterprise" assumptions should be tested before being treated as blockers.

N.03

N.04

69 pages have low text-HTML ratio

Warning 🚫 Won't Fix
About this issue. Semrush compares the character count of visible text against total HTML bytes; if text is under a threshold fraction of HTML, the page gets flagged.

This is a Webflow framework overhead issue, not a content quality issue. The flagged pages (including blog posts with 1,500+ words) contain substantive text, but Webflow's auto-generated HTML includes heavy framework markup — wrapper divs, data attributes, inline SVG icons, responsive-hidden duplicate markup — that inflates the denominator. A typical blog post has ~10–12 KB of text inside a ~250–300 KB rendered HTML page, which comes out to roughly a 4% ratio.

Text-to-HTML ratio is not a Google ranking signal — it's a Semrush heuristic carried over from early-2010s SEO — and the fix Semrush recommends ("split text and code into separate files") isn't applicable to Webflow-hosted sites. Core Web Vitals pass on both desktop and mobile per the April 22 PageSpeed Insights data: real user performance is good; this flag is informational noise.

N.05

49 pages have duplicate H1 and title tags

Warning Cascade from W.02 ✅ Resolved
About this issue. When H1 and title tag render identical text, Google loses the "same topic, different words" signal — both elements should reinforce the same subject using different phrasing.

This was the cascade effect of the April 21 H1-addition fix (see W.02 · 48 pages don't have an H1). Adding H1 bound to the post name, while the title tag was also bound to the post name (minus brand suffix), caused the two fields to render character-for-character identical text on every blog post.

Resolved on April 22 by re-adding the " | Move Concierge" brand suffix to the Blog Template title tag, which textually differentiates title from H1 while keeping them topically aligned. This matches Google's own guidance on title tags: reinforce the H1's topic with different text rather than duplicating it.

Cascade narrative

Removing the brand suffix to shorten titles (W.03) caused H1 and title to render identically (this issue). Re-adding the suffix resolves this issue but re-introduces the length issue (now W.06 equivalent) on 9 posts. The net trade — accepted length warnings on 9 posts in exchange for clearing 49 duplicate-H1/title warnings — is a clear win.

L.02

Notice ⏸️ Deferred
About this issue. Semrush flags any external URL that returned 403 (Forbidden) during its crawl.

The majority of these trace back to Webflow's own CDN-hosted placeholder.svg asset, rendered on blog pages wherever the author avatar element has a null author reference (42 of 49 blog posts have null author refs). Beyond the audit findings

Proactive cleanup done alongside the Semrush remediation.

Several sitemap hygiene and content-quality improvements were made based on discussions during the audit cycle. These weren't flagged by Semrush but materially improve crawl quality and professionalism.

Team member bulk noindex -145 URLs

Individual team member bio pages are thin, template-driven content that doesn't serve search intent. Toggled Sitemap Indexing OFF on 146 of 151 non-leadership team CMS items; kept ON for 5 keepers (Gabe Abshire, Jason Lindwall, Kevin Hockenjos, Nate Ofe, Ryan Miner).

Utility / funnel page exclusion -7 URLs

Toggled Sitemap Indexing OFF on /styleguide, /test, /thank-you, /success, /page-not-found, /schedule, /schedule-m. Internal/funnel pages that should never appear in search results.

Campaign landing page exclusion -8 URLs

Toggled Sitemap Indexing OFF on all /campaigns/dtc-* items. Paid-acquisition landing pages should not compete with organic landing pages for the same search intent.

Lorem Ipsum component removed 9 pages

Deleted the Audo Sliding Panels placeholder component instance from the Services Template. The component wasn't visible to visitors, but its Lorem Ipsum copy was still present in the page source on all 9 service pages — meaning search engines were crawling placeholder text as part of the site's content. Removing it cleans up the HTML end-to-end, which matters more for professionalism signals than any single Semrush flag.

Google Search Console sitemap submission first time

Submitted the cleaned-up 72-URL sitemap to GSC. This is the first time the moveconcierge.com sitemap has ever been submitted to Search Console — no prior SEO work had established this baseline. Unlocks Coverage report (which URLs indexed), Performance report (real search impressions/clicks), and manual indexing requests.

Webflow auto-sitemap enabled platform setting

Turning on auto-generate sitemap also causes Webflow to auto-inject a Sitemap: directive into the platform-managed robots.txt. Side-effect resolved the robots.txt warning that was previously classified as blocked.
